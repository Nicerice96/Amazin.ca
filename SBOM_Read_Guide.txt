SBOM INFO
Describes the overall SBOM document itself.

$schema – Defines the CycloneDX schema version used (here 1.6) (i.e. the CycloneDX format).

bomFormat – The SBOM format (CycloneDX).

specVersion – The version of the CycloneDX specification.

serialNumber – A unique identifier (UUID) for this SBOM instance.

version – The SBOM document version number.

METADATA
Contains information about when, how, and by which tool the SBOM was created.

timestamp – The date and time the SBOM was generated.

tools – Lists the software or tool used to create the SBOM (e.g., Trivy, with name, version, and manufacturer).

component – Describes the main/root project being analyzed (often represented by “.” for the current project).

properties – Custom metadata or tool-specific fields (e.g., schema version used by the generator).

COMPONENTS
Lists all detected software packages and dependencies used by the project.
Each component may include:

bom-ref – Unique reference for this component within the SBOM.

type – The kind of item (e.g., application, library, framework).

group / name / version – Package identifiers (for Maven, these map to GAV coordinates).

licenses – Licenses that apply to the component.

purl – Package URL (PURL) uniquely identifying the component in repositories.

properties – Extra data, often added by the SBOM tool (e.g., internal package IDs, package type).
